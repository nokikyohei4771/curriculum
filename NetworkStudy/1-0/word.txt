ネットワーク
　
　・LAN　→　限られた範囲内にあるコンピュータなどをケーブルなどで接続し、相互にデータ通信できるようにしたネットワーク

　　・有線LAN　→　構内ネットワークのうち、信号の伝送媒体として通信ケーブルを用いるもの

　　・無線LAN　→　無線通信により複数の機器間でデータの送受信を行う構内ネットワーク


　・WAN　→　地理的に離れた地点間を結ぶ通信ネットワーク

　　・インターネット　→　膨大な数のコンピュータや通信機器を相互につないだコンピュータネットワーク


ネットワーク構成

　・ノード　→　網状構造の構成要素

　・リンク　→　構成要素間を結びつける経路

　・トポロジ　→　ネットワークを構成する機器同士がどのような規則性に基づいてつながれているかを模式的に表したもの。

　　・バス型トポロジ　→　共用の通信路にすべてのノードを接続する

　　・スター型トポロジ　→　集線装置などを中心に各ノードを接続する

　　・メッシュ型トポロジ　→　各ノードが一つ以上の他のノードと接続する

　・帯域　→　通信などに用いる周波数の範囲

　　・bps　→ データ伝送速度の単位で、1秒間に何ビットのデータを送れるかを表す

　・輻輳　→　想定を超える接続要求や伝送要求が行われ、通信通話できなくなる状態


　・コリジョン　→　同じ伝送路を共有する複数の装置が同時に信号を発信して混信してしまう現象

　・ドメイン　→　TCP/IPネットワーク上で機器やネットワークを識別するドメイン名

　・ポート　→　機器やソフトウェアが外部の別の主体と接続・通信するための末端部分


モデル

　・プロトコル　→　滞りなく信号やデータ、情報を相互に伝送できるようあらかじめ決められた約束事や手順の集合
　
　・OSI参照モデル　→　データ通信を行うために機器やソフトウェア、通信規約などが持つべき機能や仕様を複数の階層に分割・整理したモデル

　　・ヘッダ　→　データや文書の本体の先頭に付け加えられるそのデータや文書自体についての情報を記述した部分

　　・カプセル化　→　オブジェクト指向プログラミングにおいて、オブジェクトとして一つの単位にまとめ、外部に対して必要な情報や手続きのみを提供すること

　　・非カプセル化　→　ヘッダを外してデータ部分を取り出すこと

　　・PDU(Protocol Data Unit)　→　プロトコルが扱うひとまとまりのデータの送受信単位

　　・MTU(Maximum Transmission Unit)　→　通信機器などが一度に送信できる最大のデータ量

　　・フレーム　→　表示領域を縦または横に複数の領域に分割してそれぞれに別のページを表示したもの

　　・パケット　→　データをある長さごとに区切り、送信元や宛先などの制御情報を付加した小さなまとまり

　　・セグメント　→　全体をいくつかに分割したうちの断片や区分

　・TCP/IPモデル　→IPを中心とする,よりシンプルな4階層のモデル


物理層

　・ビット　→　情報量の最小単位
　
　・ツイストペアケーブル　→　電線を２本1組としてらせん状により合わせ、ゴムなどの被覆材で覆って一本のケーブルとしたもの

　　・UTP(Unshielded Twisted Pair cable)　→　ツイストペアケーブルで電磁ノイズを遮断するシールド加工を行っていないもの

　　・STP(Shielded Twisted Pair cable) →　ツイストペアケーブルで電磁ノイズを遮断するシールド加工を行ったもの

　　・カテゴリー　→　ツイストペアケーブルの標準規格でケーブルの電気特性グレード

　　・コネクタ　→　他の部品に簡単に着脱できるように設けられた部品

　・ストレートケーブル　→　内部に複数の芯線が通っている通信ケーブルのうち、各芯線が両端で同じ位置のピンに結線されているもの

　・クロスケーブル　→　内部に複数の芯線が通っている通信ケーブルのうち、一部の芯線が両端で別の位置のピンに結線されているもの

　・光ファイバーケーブル　→　光ファイバーを用いた通信ケーブル

　・ハブ　→　複数のケーブルを接続して相互に通信できるようにする集線装置、中継装置のこと

　　・半二重通信　→　双方向通信において一度に片方しか送信できず、両者が同時に送信することができない方式


・データリンク層

　・イーサネット　

　　・イーサネット　→　主に室内や建物内でコンピュータや電子機器をケーブルでつないで通信する有線LANの標準の一つ

　　・ファストイーサネット　→　イーサネットのうち、100Mbpsでの通信に対応した規格群の総称

　　・ギガビットイーサネット　→　イーサネットのうち、1Gbpsでの通信に対応した規格群の総称

　・イーサネットヘッダ　→　イーサネットフレームのヘッダ部

　・トレーラ　→　区切られたデータ一つ一つに付加される制御情報
　
　・CSMA/CD　→　単一の伝送路を複数の機器が共用する際、伝送路の使用権に調整できる通信方式

　・MACアドレス　→　各通信主体を一意に識別するために物理的に割り当てられた48ビットの識別番号

　・スイッチ　→　中継・集線装置のうち、データの宛先を見て関連する先にのみ転送を行う機能をもったもの

　　・MACアドレステーブル　→　どのポートにどのMACアドレスを持つ機器が接続されているかを記録したもの

　　・フラッディング　→　許容量を超えるデータや処理要求が届き麻痺状態に陥る現象

　　・全二重通信　→　双方向通信において、両者が同時に送信することのできる方式

　・VLAN　→　一つの構内ネットワーク内に仮想的なグループを設定し、あたかも一つのLANであるかのように運用する技術

　　・タグVLAN　→　イーサネットフレームの一部にVLANの識別番号を書き込み、同じVLANのIDを持つポート間に流通させる

　　・ポートベースVLAN　→　一台のLANスイッチのケーブル接続口ごとにどのVLANに所属するか設定する

　　・アクセスポート　→　一つのVLANのみに所属するポート

　　・トランクポート　→　複数のVLANに所属し、それぞれのデータを混在させて伝送するポート

　・DTP(DeskTop Publishing)　→　出版物や印刷物の原稿作成などの作業をコンピュータで行い、最終的に印刷可能な原稿の作成まで行う　

　・デフォルトVLAN　→　VLANの指定がないポートやフレームの通信に用いられるVLAN

　・ネイティブVLAN　→　トランクリンク上で唯一タグをつけずに送受信されるVLAN

・ブロードキャストストーム　→　一斉配信用の特殊なデータや信号が際限なく転送され続ける現象

・スパニングツリープロトコル　→　ループによる問題を回避するためのプロトコル

・リンクアグリゲーション　→　複数の回線を仮想的に一本の回線とみなすことで通信速度および耐故障性を向上させる技術

・EtherChannel　→　複数のイーサネット回線を束ねて仮想的に一本の大容量回線として利用する機能
　

・オートネゴシエーション　→　通信相手との間で対応している規格や通信モードの違いを自動的にすり合わせ、最適な設定を行うもの
